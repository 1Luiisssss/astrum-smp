01 · Memoria RAM
ASIGNACIÓN DE RAM
Mínimo
2 GB
Jugable con lag
Recomendado
4 GB
Experiencia fluida
Óptimo
6–8 GB
Sin interrupciones
⚠️
No asignes toda tu RAM. Deja al menos 4 GB libres para Windows y otros procesos. Más RAM no siempre significa más FPS.
🎮
Cómo asignar RAM en el launcher oficial
Abre el launcher → Instalaciones → edita tu perfil de Fabric → Más opciones → en "Argumentos JVM" cambia -Xmx2G por el valor que quieras.
-Xmx4G ← 4 GB -Xmx6G ← 6 GB -Xmx8G ← 8 GB
📦
En launchers alternativos (Prism, MultiMC, ATLauncher)
Ve a Configuración de la instancia → Java → Memoria máxima. Estos launchers tienen una interfaz visual más clara para cambiar esto.
⚙️
02 · Instalación
INSTALAR FABRIC
ℹ️
Astrum SMP corre en Fabric 1.21. Asegúrate de instalar exactamente esa versión para evitar incompatibilidades.
01
Descarga el instalador de Fabric
Ve a fabricmc.net/use/installer y descarga el instalador para tu sistema operativo.
02
Ejecuta el instalador
Abre el archivo descargado. Selecciona la versión 1.21 de Minecraft y haz clic en Instalar. El loader version déjalo en la más reciente.
03
Agrega Fabric API
Descarga Fabric API desde Modrinth y colócala en tu carpeta .minecraft/mods/. Es requisito para la mayoría de mods.
04
Lanza el juego
En el launcher selecciona el perfil fabric-loader-1.21 y lanza. Verás "Fabric" en el menú principal si todo salió bien.
⚡
03 · Java
ARGUMENTOS JVM
🚀
Argumentos recomendados (pega esto en tu launcher)
Estos flags mejoran el garbage collector de Java y reducen los micro-stutters que se sienten como lag repentino.
-Xmx4G -Xms4G -XX:+UseG1GC -XX:+ParallelRefProcEnabled -XX:MaxGCPauseMillis=200 -XX:+UnlockExperimentalVMOptions -XX:+DisableExplicitGC -XX:+AlwaysPreTouch -XX:G1HeapWastePercent=5 -XX:G1MixedGCCountTarget=4 -XX:G1MixedGCLiveThresholdPercent=90 -XX:G1RSetUpdatingPauseTimePercent=5 -XX:SurvivorRatio=32 -XX:+PerfDisableSharedMem -XX:MaxTenuringThreshold=1
✅
Usa Java 21. Es la versión más rápida para Minecraft 1.21. Descárgala en adoptium.net. Evita Java 8 en versiones modernas.

05 · Gráficos
OPCIONES DE VIDEO
📊
Configuración para máximo rendimiento
En Opciones → Video (con Sodium instalado):
Distancia de renderizado: 8–12 chunks Calidad de gráficos: Rápido Nubes: Desactivadas Partículas: Mínimas VSync: Desactivado (activa FPS limit manual) Renderizado de entidades: 4 chunks Smooth lighting: Apagado (opcional)
🎯
Límite de FPS
No dejes el FPS ilimitado. Ponlo en el doble de tu tasa de refresco (ej: 144Hz → límite 288). Esto reduce el uso de CPU/GPU y el calor sin afectar la experiencia.
🌐
06 · Conectividad
OPTIMIZACIÓN DE RED
🔌
Usa cable ethernet
El WiFi introduce jitter (variación de latencia) que se siente como rubber-banding aunque el ping sea bajo. Un cable reduce el lag perceptible aunque el ping sea el mismo número.
📡
Cambia el DNS
Usar DNS lentos puede afectar la resolución inicial. Configura 1.1.1.1 (Cloudflare) o 8.8.8.8 (Google) en tu adaptador de red.
⚡
Cierra aplicaciones en segundo plano
Discord en streaming, Chrome con muchas pestañas, y actualizaciones de Windows consumen ancho de banda. Ciérralos antes de jugar si tienes conexión limitada.

08 · Troubleshooting
PROBLEMAS COMUNES
❌
El juego crashea al abrir
Probablemente un mod incompatible. Revisa el crash report en .minecraft/crash-reports/. Busca la línea "Caused by:" y ese es el mod conflictivo. Elimínalo o actualízalo.
🐌
Stutters cada pocos segundos
Es garbage collection de Java. Solución: asigna RAM fija (Xms = Xmx) y usa los argumentos JVM de la sección 03. El mod FerriteCore también ayuda.
🔴
Ping alto solo en Astrum SMP
El servidor está en Latinoamérica. Si estás fuera de la región el ping será mayor. Prueba con una VPN a Chile/Argentina para verificar si el problema es de enrutamiento.
⚠️
Out of Memory error
Estás asignando poca RAM o demasiada. Si tienes 8GB de sistema, asigna 4GB a Minecraft. Si tienes 16GB, asigna 6-8GB. Nunca asignes más del 75% de tu RAM total.
